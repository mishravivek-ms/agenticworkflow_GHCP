# Plan: Basic Authentication Flow

## Overview

Build a full-stack application with a login UI and a REST API backend that authenticates a user against hardcoded credentials.

**Tech Stack**
- Backend: FastAPI (Python 3.11+)
- Frontend: FastUI
- Package Manager: uv
- API Style: REST

---

## Project Structure

```
project/
├── pyproject.toml          # uv-managed project manifest
├── uv.lock                 # lockfile (auto-generated by uv)
├── README.md
├── backend/
│   ├── __init__.py
│   ├── main.py             # FastAPI app entry-point (also serves FastUI)
│   ├── auth.py             # Authentication logic
│   └── models.py           # Pydantic request/response models
└── frontend/
    ├── __init__.py
    └── ui.py               # FastUI page & form definitions
```

### 2. Backend – FastAPI Application

#### 2.1 Pydantic Models (`backend/models.py`)

- [ ] Define `LoginRequest` model:
  ```python
  class LoginRequest(BaseModel):
      username: str
      password: str
  ```
- [ ] Define `LoginResponse` model:
  ```python
  class LoginResponse(BaseModel):
      success: bool
      message: str
  ```

#### 2.2 Authentication Logic (`backend/auth.py`)

- [ ] Define hardcoded credentials as module-level constants:
  ```python
  HARDCODED_USERNAME = "username"
  HARDCODED_PASSWORD = "passcode"
  ```
- [ ] Implement `authenticate(username: str, password: str) -> bool`:
  - Returns `True` only when both `username` and `password` match the constants exactly (case-sensitive).
  - Uses constant-time comparison (`hmac.compare_digest`) to avoid timing attacks.

#### 2.3 REST API Endpoints (`backend/main.py`)

- [ ] Create FastAPI app instance.
- [ ] Mount FastUI static assets (required by FastUI).
- [ ] `POST /api/login`
  - Accepts `LoginRequest` JSON body.
  - Calls `authenticate()`.
  - Returns `LoginResponse`:
    - `{"success": true, "message": "Login successful"}` on match.
    - `{"success": false, "message": "Invalid username or password"}` on mismatch (HTTP 401).
- [ ] `GET /` – serve the FastUI login page (see Frontend section).
- [ ] `GET /api/` – FastUI JSON route that returns the page component tree.
- [ ] Configure CORS if frontend is served from a different origin (optional for dev).

---

### 3. Frontend – FastUI Login Page (`frontend/ui.py`)

FastUI renders a React app driven entirely by Python-defined components.

- [ ] Define a `login_page()` function that returns a `Page` containing:
  - A `Heading` component: `"Login"`
  - A `Form` component targeting `POST /api/login` with:
    - `InputField` for `username` (label: "Username", required: true)
    - `InputField` for `password` (input type: `password`, label: "Password", required: true)
    - Submit button label: `"Sign In"`
  - A conditional `Paragraph` / `Alert` component to display success or error message after form submission (handled via FastUI's built-in form response rendering).
- [ ] Register the page route in `backend/main.py`:
  ```python
  from frontend.ui import login_page

  @app.get("/api/", response_model=FastUI, response_model_exclude_none=True)
  def fastui_root() -> list[AnyComponent]:
      return login_page()
  ```

### 5. Testing

All tests live under `tests/` and use `pytest` + `httpx`.

#### 5.1 Authentication Unit Tests (`tests/test_auth.py`)

- [ ] Test `authenticate("username", "passcode")` → `True`
- [ ] Test wrong password → `False`
- [ ] Test wrong username → `False`
- [ ] Test both wrong → `False`
- [ ] Test empty strings → `False`

#### 5.2 API Integration Tests (`tests/test_api.py`)

- [ ] `POST /api/login` with correct credentials → HTTP 200, `success: true`
- [ ] `POST /api/login` with wrong password → HTTP 401, `success: false`
- [ ] `POST /api/login` with wrong username → HTTP 401, `success: false`
- [ ] `POST /api/login` with missing fields → HTTP 422 (validation error)
- [ ] `GET /` → HTTP 200 (FastUI HTML shell)
- [ ] `GET /api/` → HTTP 200, JSON list of FastUI components

#### 5.3 Running Tests

```bash
uv run pytest tests/ -v
```

---

### 6. Database

No database is required for this feature. Credentials are hardcoded in `backend/auth.py` as specified in the issue. If credentials need to be persisted or managed in the future, the recommendation is to add SQLite (via `SQLModel` or `SQLAlchemy`) with a `users` table.

---

### 7. Security Considerations

- [ ] Use `hmac.compare_digest` for credential comparison to prevent timing attacks.
- [ ] Return a generic error message (`"Invalid username or password"`) rather than indicating which field was wrong.
- [ ] Do **not** log raw passwords anywhere.
- [ ] Return HTTP 401 (not 200) on failed authentication.

---

### 8. Acceptance Criteria

- [ ] Navigating to `http://localhost:8000/` shows a login form with Username and Password fields and a Sign In button.
- [ ] Submitting `username` / `passcode` shows a success message.
- [ ] Submitting any other combination shows an error message.
- [ ] All unit and integration tests pass.
- [ ] Application starts with a single `uv run uvicorn backend.main:app --reload` command.
